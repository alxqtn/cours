# Titre principal (H1 — un seul par document)

Petit paragraphe d'introduction. On laisse **une ligne vide** entre les blocs
pour que le rendu sépare bien les éléments.

## Titre de section (H2)

### Sous-section (H3)

## Emphase

Du texte en *italique* (ou _italique_), en **gras**, en ***gras italique***,
et en ~~barré~~.

## Listes

Liste à puces :

- Premier élément
- Deuxième élément
  - Sous-élément (indenté de 2 espaces)
  - Autre sous-élément
- Troisième élément

Liste numérotée (l'ordre compte) :

1. Première étape
2. Deuxième étape
3. Troisième étape

## Code

Du code `inline` avec des backticks simples (noms de fichiers, commandes…).

Un bloc de code avec le langage précisé pour la coloration :

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Bonjour");
    }
}
```

## Liens et images

Un [lien nommé](https://example.com) plutôt qu'une URL brute.

Une image : ![texte alternatif](chemin/vers/image.png)

## Citation

J'écris ici.
Et là.

> Ceci est une citation.

> On peut la mettre sur plusieurs lignes.

## Tableau

| Colonne A | Colonne B | Colonne C |
|-----------|:---------:|----------:|
| gauche    | centré    | droite    |
| valeur 1  | valeur 2  | valeur 3  |

Les `:` dans la ligne de séparation gèrent l'alignement
(gauche par défaut, `:---:` centré, `---:` à droite).

## Divers

Une ligne horizontale de séparation :

---

Une tâche à cocher (case à cocher) :

- [x] Tâche faite
- [ ] Tâche à faire

Un saut de ligne forcé dans un paragraphe se fait avec deux espaces
en fin de ligne, ou une ligne vide pour un nouveau paragraphe.
